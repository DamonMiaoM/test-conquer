# 判断题闪卡 Web App 设计文档

## 概述

为 Test Conquer 在线模拟考试平台构建一个前端闪卡 Web App，支持 300 道人工智能训练师（三级）判断题的交互式练习。

## 核心需求

1. 300 道判断题，逐题展示
2. 用户点击"对"/"错"按钮作答
3. 即刻显示正确/错误反馈
4. 显示简要知识点解释 + 可跳转学习链接
5. 支持随机出题 / 顺序出题切换
6. 错题记录与错题回顾
7. **Tab 导航框架**：判断题 / 选择题 / 操作题 三个 Tab，判断题上线，其余显示"开发中"
8. 键盘快捷键支持（← 错，→ 对，Space 下一题）
9. 部署到 GitHub Pages，方便分享

## 技术方案

- **单文件方案**：一个 `index.html`，内嵌 CSS + JS
- **数据**：300 道题 + 答案内嵌为 JSON 数组
- **无后端**：纯前端，localStorage 存储错题记录
- **部署**：GitHub Pages

## 交互细节

### Tab 导航
- 三个 Tab：`判断题` | `选择题` | `操作题`
- 当前 Tab 高亮（底部边框或背景色区分）
- 选择题/操作题 Tab 显示"开发中，敬请期待"占位页
- 切换 Tab 不丢失判断题进度

### 答题流程
1. 显示题目 + 两个按钮
2. 点击按钮 → 按钮禁用，显示反馈
3. 反馈区展示：结果（对/错）、正确答案、知识点解释、链接
4. 点击"下一题" → 刷新题目，恢复按钮

### 随机模式
- 默认随机顺序
- 切换开关改变出题顺序
- 随机模式下不重复已答题目（直到全部答完）

### 错题记录
- 答错的题自动存入 localStorage
- "错题本"按钮弹出错题列表
- 错题可点击重做
- 可清空错题记录

### 键盘快捷键
- `←` 左箭头：选"错"
- `→` 右箭头：选"对"
- `Space` 空格：下一题
- 仅在可答题状态下有效

## 文件结构

```
src/
  index.html          ← 单文件，包含所有 CSS + JS + 数据
data/
  questions.json      ← 原始题库数据（开发用，不部署）
```

## 部署

- GitHub Pages 直接从 `src/` 目录部署
- 或将 `index.html` 放到项目根目录
- 确保不暴露 `data/questions.json` 原始数据

## 非目标（Out of Scope）

- 用户登录/注册
- 云端进度同步
- 选择题/操作题的实际内容
- 移动端 App
- 后端服务
